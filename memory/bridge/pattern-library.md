# Decision Pattern Library

*Recurring decision types and the frameworks that worked. Portable across roles.*

## "Grantee Wants to Pivot Mid-Grant"
- Assess: is the pivot toward or away from the portfolio thesis?
- Check: does the revised scope still justify the budget?
- Require: written revised scope with updated milestones
- Default: allow if thesis-aligned and team is capable. Rigidity kills innovation.

## "Two Competing Standards — Which Do We Support?"
- Default: support interoperability/bridging, not a winner
- Exception: if one standard is clearly dying, don't prolong it
- Key question: which approach has the implementors, not just the spec writers?

## "Proposal Overlaps With Existing Grant"
- Assess complementarity vs. redundancy — same problem, different approach is OK
- Check: would the PIs collaborate or compete?
- If redundant: decline with clear rationale and referral
- If complementary: fund with explicit coordination requirement

## "Co-Funder Interested But Different Theory of Change"
- Find shared metrics first — if you can't agree on what success looks like, don't co-fund
- Maintain independent evaluation even in co-funded work
- Written MOU on decision rights before money moves
