# Reusable Skill: Multi-Perspective Evaluation

A framework for evaluating proposals, projects, or decisions from multiple angles simultaneously. Learned and refined through grant proposal evaluation but applicable to any complex assessment.

## The Pattern
1. Technical assessment — is the approach sound?
2. Equity assessment — who benefits, who's excluded?
3. Operational assessment — is the budget/team/timeline realistic?
4. Ecosystem assessment — does this fill a gap or duplicate?
5. Synthesize into a Decision Frame, never a recommendation

## Lessons Learned
- The equity perspective must be substantive, not a section you fill in after the real analysis
- "Who bears the cost of this decision?" is the most important equity question
- Technical brilliance doesn't compensate for team risk — a great architecture with a single-point-of-failure team is a bad investment
- Always check for duplication before assessing merit — the best proposal in the world is a bad investment if someone else is already doing it
